# 视觉验收标准

日期：2026-06-26

## 固定条件

- 原版静态页面只从 `http://127.0.0.1:8000/` 截图。
- 迁移版 uni-app H5 只从 `http://127.0.0.1:5174/` 截图。
- 迁移版默认使用 `VITE_API_MODE=mock`。
- 移动视口必须至少覆盖 `390x844` 和 `519x927`。
- 截图固定在首屏 `scrollY=0`，等待页面网络空闲后再等待稳定时间。

## 判定规则

通过结论只能写作：`通过：迁移版与原版截图肉眼无明显区别`。

不通过结论只能写作：`不通过：仍存在明显视觉差异`。

以下情况必须判定不通过：

- 主视觉资产、背景图或 hero 构图明显不同。
- 顶部导航、底部导航或主要入口结构不同。
- 主题色、卡片风格、圆角、阴影、图标风格明显不同。
- 首屏信息层级或核心业务状态不同。
- 关键操作区、固定底栏、二维码/票码等核心视觉中心缺失。

允许差异：

- 系统字体渲染、抗锯齿和 1-2px 级间距差异。
- 小程序平台限制导致的极小组件渲染差异，但必须保留原版视觉语言。

## 后续 Phase 引用要求

- 每个 Phase 必须复用本标准，不得用构建通过、测试通过或旧截图目录替代视觉结论。
- 每个页面必须保留原版截图、修复前迁移版截图、修复后迁移版截图、差异关闭记录和人工视觉结论。
- 若任一截图普通用户一眼能看出是另一套设计，当前 Phase 继续修复，不得标记 completed。
