# Phase 1 视觉差异清单

日期：2026-06-26

等级定义：

- P0：普通用户一眼能看出不是同一套设计。
- P1：布局、主色、关键资产、卡片或导航明显不一致。
- P2：局部间距、字体、阴影、图标细节偏差。

## 首页

结论：`不通过：仍存在明显视觉差异`

- P0：原版首屏是水彩风珠海湾区大插画背景，迁移版是夜景邮轮照片卡片，主视觉资产完全不同。
- P0：原版顶部品牌、搜索框、问候语和轮播标题叠加在 hero 上；迁移版是系统导航标题 + 独立 hero 卡 + hero 下方搜索框。
- P1：原版底部导航为“首页 / 客服 / 我的”，迁移版为“首页 / 订单 / 我的”，导航结构和选中态不一致。
- P1：原版热门目的地是浅色彩块图标卡片，迁移版是两列白底边框卡片，圆角、图标、色彩和密度不同。
- P1：原版促销船票卡在首屏底部露出，迁移版首屏露出的是渐变入口底部，信息层级不同。

## 我的

结论：`不通过：仍存在明显视觉差异`

- P0：原版来自 `index.html` 内“我的”页，使用头像图标、已实名状态、下一段行程、票券 KPI 和服务宫格；迁移版是简化标题、资料卡和普通服务卡。
- P1：原版服务入口含票券夹、待使用、退款中、我的订单等明确卡片层级；迁移版入口文案和卡片结构不同。
- P2：字体大小、图标风格、卡片阴影和间距均未贴合原版。

## 订单

结论：`不通过：仍存在明显视觉差异`

- P0：原版订单页是 `index.html` 内状态页，含返回我的按钮、状态筛选和多张订单卡；迁移版页面头部、说明文案和票券基础壳改变了视觉结构。
- P1：原版筛选项为“全部 / 待使用 / 已预约 / 已完成 / 退款售后”，迁移版加入“待支付 / 已退款”等状态且样式不同。
- P1：原版订单卡以紧凑 meta 栅格和右侧状态标签呈现，迁移版组件卡片样式、按钮和间距不同。

## 票券

结论：`不通过：仍存在明显视觉差异`

- P0：原版首屏核心是“待使用”状态、核销横幅、大二维码和底部操作按钮；迁移版是普通票券摘要卡，没有同等二维码核销视觉中心。
- P1：原版有手机状态栏式顶部和多图标导航，迁移版使用 uni-app 默认导航栏。
- P1：原版底部固定操作为“景区电话 / 申请退款”，迁移版首屏未对齐该底部操作区。

## 环岛游详情

结论：`不通过：仍存在明显视觉差异`

- P0：原版详情页是深色沉浸式 hero、视频/媒体、浮动导航 chip 和底部购买栏；迁移版为浅色页面、步骤 tab 和普通卡片。
- P1：原版信息卡、推荐区、图文内容和注意事项使用深色海上夜游视觉语言；迁移版用统一白底卡片和绿色按钮。
- P2：局部资产虽然同为夜景邮轮，但裁切、遮罩、文字层级和背景色不同。

## 环岛游订票

结论：`不通过：仍存在明显视觉差异`

- P0：原版订票页是独立手机壳流程，首屏含顶部深色导航、视频卡、深色 hero copy、指标卡和固定底部栏；迁移版仍是浅色步骤 tab 页面。
- P1：原版“近期推荐 / 班次查询 / 可售班次”视觉区块、字段样式和底栏价格区未被还原。
- P1：迁移版 `island-detail` 与 `island-booking` 当前映射到同一 detail 状态，Phase 4 需补齐 traveler/pay/ticket 状态对比。

## 司机端

结论：`不通过：仍存在明显视觉差异`

- P0：原版默认是司机登录/注册首屏，带车内背景图、返回首页/司机入驻 chip、三步注册卡和登录表单；迁移版默认进入工作台信息流。
- P1：原版 mode tabs 是“司机登录 / 司机注册”双 tab，迁移版是“登录 / 注册 / 工作台”三 tab。
- P1：原版表单输入、按钮、背景色和卡片位置均未对齐；迁移版二维码/钱包内容提前出现在首屏。

## 后续修复顺序

1. Phase 2：首页首屏和首页主体，先关闭 P0。
2. Phase 3：我的、订单、票券基础页。
3. Phase 4：环岛游详情/订票流程和司机端。
4. Phase 5：全量最终截图复验。
